# 05. 학습자용 업무 적용 가이드

## 이 예제를 실제 업무로 바꾸는 핵심

이 예제의 핵심은 `좋은 프롬프트`에서 멈추지 않고 `반복 실행 가능한 자동화 구조`로 바꾸는 것이다.

업무 적용 시에는 아래 4가지를 먼저 정해야 한다.

1. 어떤 폴더를 입력 폴더로 볼 것인가
2. 최신 파일을 어떤 규칙으로 고를 것인가
3. 누가 언제 실행할 것인가
4. 결과 파일을 어디에 저장할 것인가

## 추천 학습 순서

처음부터 정답 절차만 따라하게 하기보다 아래 순서를 추천한다.

### 1. 자유 탐색 먼저

학습자에게 먼저 아래 질문을 던진다.

1. 프롬프트만 있으면 자동화라고 볼 수 있을까
2. 최신 파일 3개는 어떻게 고를까
3. 사용자는 어떤 방식으로 실행해야 편할까
4. 결과 파일은 어디에 저장돼야 할까
5. 사용자가 미리 맞춰야 하는 규칙은 무엇일까

이 단계에서는 정답보다 생각의 틀을 만드는 것이 중요하다.

### 2. 단계별 실습 나중

그 다음 아래 순서대로 실제 실행을 따라가게 한다.

1. Node.js 설치 확인
2. 파일명 규칙 확인
3. `samples` 폴더에 파일 3개 넣기
4. `실험데이터_통합_실행.bat` 더블클릭
5. `outputs` 폴더의 결과 파일 확인
6. `검토필요` 시트 확인

즉 `생각은 자유롭게, 실행은 단계적으로`가 가장 무난하다.

## 먼저 알아둘 것: Node.js란?

이 자동화 예제는 `Python`이 아니라 `Node.js`로 실행된다.

Node.js는 자바스크립트 파일을 컴퓨터에서 실행할 수 있게 해주는 실행 프로그램이다.  
이 폴더의 [merge_experiment_results.mjs](C:\Users\user\Google 드라이브\03_GPT교안\코젠바이오텍_바이브코딩_자동화\10_자동화_실험데이터_통합\merge_experiment_results.mjs)가 바로 Node.js로 실행되는 파일이다.

쉽게 말하면 역할은 아래와 같다.

- `실험데이터_통합_실행.bat`: 사용자가 더블클릭하는 버튼
- `run_latest_experiment_merge.ps1`: 중간 실행 스크립트
- `merge_experiment_results.mjs`: 실제 병합 작업을 하는 코드
- `Node.js`: 이 코드를 실행해 주는 프로그램

즉 Node.js가 없으면 BAT 파일을 눌러도 실제 병합 코드가 실행되지 않는다.

## Node.js 설치 방법

### 가장 쉬운 방법

1. 웹브라우저에서 [Node.js 공식 사이트](https://nodejs.org/)에 접속한다.
2. `LTS` 버전을 다운로드한다.
3. 설치 파일을 실행한다.
4. 설치 과정에서 기본 옵션으로 `Next`를 눌러 진행한다.
5. 설치가 끝나면 PowerShell이나 명령 프롬프트를 새로 연다.

교육용 가이드에서는 보통 `LTS 버전 설치`라고 안내하는 것이 가장 무난하다.

## 설치 확인 방법

PowerShell 또는 명령 프롬프트에서 아래 명령을 실행한다.

```powershell
node --version
```

버전 번호가 나오면 정상 설치된 것이다.  
예: `v22.14.0`

아무 것도 나오지 않거나 `node`를 찾을 수 없다는 메시지가 나오면 설치가 안 되었거나, 설치 후 창을 다시 열지 않은 경우가 많다.

## 사전 준비 규칙

자동화가 안정적으로 동작하려면 사용자가 아래 규칙을 먼저 맞춰야 한다.

### 1. 입력 폴더 규칙

- 통합 대상 파일은 모두 `samples` 폴더에 넣는다.
- 통합과 무관한 임시 파일은 같은 폴더에 두지 않는다.
- 이전 테스트 파일이 남아 있으면 최신 파일 선택에 영향을 줄 수 있으므로 정리한다.

### 2. 파일명 규칙

- 파일명에는 날짜와 장비명이 들어가도록 맞춘다.
- 예시:
  - `실험데이터_20260421_장비A.csv`
  - `실험데이터_20260421_장비B.xlsx`
  - `실험데이터_20260421_장비C.xlsx`
- 날짜는 가능하면 `YYYYMMDD` 형식으로 통일한다.
- 장비명은 `장비A`, `장비B`, `장비C`처럼 일관되게 유지한다.

### 3. 업로드 규칙

- 같은 날짜 기준으로 통합할 파일 3개를 모두 넣은 다음 실행한다.
- 파일 1개만 먼저 넣고 중간에 실행하지 않는다.
- 파일을 수정해 다시 저장했다면 최신 수정 시간이 바뀌므로, 의도한 파일이 맞는지 한 번 확인한다.

### 4. 데이터 구조 규칙

- 각 파일에는 공통 7개 표준 항목에 대응되는 컬럼이 있어야 한다.
- 컬럼명은 달라도 되지만, 매핑 가능한 수준이어야 한다.
- 완전히 새로운 컬럼명이 들어오면 매핑 규칙을 먼저 추가해야 한다.
- 빈 파일이나 헤더만 있는 파일은 입력 대상에서 제외한다.

### 5. 날짜 기준 규칙

- 샘플 파일명 날짜와 실제 업무 날짜 기준을 맞춘다.
- 교육용 시연에서는 파일명 날짜와 출력 파일 날짜가 같도록 맞추면 이해가 쉽다.
- 사용자가 “오늘 데이터”라고 이해하도록 하려면 파일명 예시도 당일 기준으로 제시하는 것이 좋다.

### 6. 결과 확인 규칙

- 실행 후에는 `outputs` 폴더에 결과 파일이 생성됐는지 확인한다.
- `검토필요` 시트가 비어 있지 않다면 누락 컬럼이나 매핑 실패를 먼저 본다.
- 자동화 결과는 최종 배포 전에 담당자가 한 번 검토하는 절차를 둔다.

## 가장 쉬운 실행 방식

사용자 편의를 우선하면 PowerShell 명령 입력보다 BAT 파일 더블클릭이 가장 쉽다.

- [실험데이터_통합_실행.bat](C:\Users\user\Google 드라이브\03_GPT교안\코젠바이오텍_바이브코딩_자동화\10_자동화_실험데이터_통합\실험데이터_통합_실행.bat)을 더블클릭한다.
- 내부적으로 [run_latest_experiment_merge.ps1](C:\Users\user\Google 드라이브\03_GPT교안\코젠바이오텍_바이브코딩_자동화\10_자동화_실험데이터_통합\run_latest_experiment_merge.ps1)을 호출한다.
- 실제 병합은 [merge_experiment_results.mjs](C:\Users\user\Google 드라이브\03_GPT교안\코젠바이오텍_바이브코딩_자동화\10_자동화_실험데이터_통합\merge_experiment_results.mjs)가 수행한다.

즉 사용자는 보통 BAT 파일만 보면 된다.

## 추천 운영 방식

### 방식 1. 사용자가 BAT 파일을 직접 실행

가장 쉬운 시작 방식이다.

- 실험 파일 3개를 `samples` 폴더에 넣는다.
- `실험데이터_통합_실행.bat`를 더블클릭한다.
- 결과는 `outputs` 폴더에 날짜형 파일명으로 저장된다.

이 방식은 교육 초반이나 반자동 업무에 적합하다.

### 방식 2. 매일 자동 실행

일정이 정해져 있다면 Windows 작업 스케줄러에 아래 파일을 등록한다.

- 1순위: `실험데이터_통합_실행.bat`
- 2순위: `run_latest_experiment_merge.ps1`

예시:

- 평일 오후 6시 실행
- `samples` 폴더에서 최신 파일 3개 선택
- 통합 결과를 `outputs` 폴더에 저장

이 방식은 사용자가 매번 실행 버튼을 누르지 않아도 되는 구조다.

## 현재 스크립트가 하는 일

`merge_experiment_results.mjs`는 아래 순서로 동작한다.

1. `samples` 폴더에서 최신 `csv/xlsx` 3개를 찾는다.
2. 파일 헤더를 보고 장비 A/B/C 타입을 판별한다.
3. 장비별 컬럼명을 공통 7개 표준 항목으로 변환한다.
4. 추가 컬럼은 별도 시트로 보존한다.
5. 검토가 필요한 이슈를 별도 시트에 남긴다.
6. 결과를 `outputs/실험데이터_통합결과_YYYYMMDD.xlsx`로 저장한다.

## 학습자가 실무에 맞게 바꿔야 할 것

실무에서는 아래 항목을 조직 상황에 맞게 조정하면 된다.

- 입력 폴더 경로
- 장비별 파일명 패턴
- 표준 컬럼 7개 정의
- 컬럼 매핑 규칙
- 결과 저장 경로
- 실행 스케줄

예를 들어 장비가 5대로 늘어나면 스크립트 안의 매핑 정의를 추가하면 된다.

## 교육 포인트

- 프롬프트는 규칙 설명서 역할을 한다.
- 스크립트는 그 규칙을 반복 실행하는 엔진이다.
- BAT 파일은 비개발자도 쉽게 누를 수 있는 실행 버튼 역할을 한다.
- Node.js는 그 엔진을 실제로 돌리는 실행 프로그램이다.
- 자동화라고 부르려면 트리거가 필요하다.
  - 수동 트리거: BAT 더블클릭
  - 자동 트리거: 스케줄 실행
- 그리고 트리거 이전에 사용자가 지켜야 하는 사전 준비 규칙이 있어야 실제 운영이 안정적이다.

즉, `프롬프트를 잘 만들었다`에서 끝내지 말고 `누가 안 눌러도 돌아가거나, 한 번만 눌러도 바로 끝나는 구조`와 `그 전에 무엇을 맞춰야 하는지`까지 함께 안내해야 한다.

```powershell

```

## 요약

학습자에게는 이렇게 설명하면 좋다.

`좋은 자동화는 프롬프트에서 끝나지 않는다. 최신 파일을 자동으로 잡아오고, BAT 파일처럼 쉬운 버튼으로 실행되며, 결과가 지정된 폴더에 저장되어야 한다. 그리고 그 전에 사용자가 파일명, 폴더, 날짜, 업로드 순서를 같은 규칙으로 맞춰야 하며, 이 자동화 예제는 Node.js 설치가 필요하다.`
