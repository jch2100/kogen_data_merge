# 07. 표준실습 상세가이드

## 문서 목적

이 문서는 학습자가 Codex를 이용해 `실험데이터 3개 파일을 자동으로 통합하고 실행 파일까지 만드는 과정`을 처음부터 끝까지 그대로 따라 할 수 있도록 만든 표준 실습 가이드다.

이 문서의 목표는 단순히 파일을 합치는 것이 아니라 아래 4가지를 한 번에 경험하게 하는 것이다.

- 자동화 문제를 구조적으로 정의하기
- Codex에 설계와 구현을 요청하기
- 결과를 실제 폴더에 저장되게 만들기
- 사용자가 `.bat` 파일로 다시 실행할 수 있게 만들기

## 실습 결과물

실습이 끝나면 아래 결과가 나와야 한다.

- `samples` 폴더의 최신 파일 3개를 읽는다.
- 파일을 하나의 xlsx로 통합한다.
- 결과를 `outputs` 폴더에 저장한다.
- 사용자는 `실험데이터_통합_실행.bat`를 더블클릭해 다시 실행할 수 있다.

## 실습 전 준비

### 1. 준비 파일

학습자는 아래 파일 3개를 준비한다.

- `csv` 또는 `xlsx` 형식의 실험 결과 파일 3개
- 가능하면 날짜와 장비명이 들어간 파일명

예시:

- `실험데이터_20260421_장비A.csv`
- `실험데이터_20260421_장비B.xlsx`
- `실험데이터_20260421_장비C.xlsx`

### 2. 폴더 준비

강사가 안내한 실습 폴더를 연다.

예시 폴더:

`C:\Users\user\Google 드라이브\03_GPT교안\코젠바이오텍_바이브코딩_자동화\10_자동화_실험데이터_통합`

### 3. Node.js 설치 확인

이 실습은 Python이 아니라 Node.js를 사용한다.

PowerShell 또는 명령 프롬프트에서 아래 명령을 실행한다.

```powershell
node --version
```

정상 예시:

```text
v22.14.0
```

버전이 보이지 않으면 Node.js 공식 사이트에서 `LTS` 버전을 설치한다.

## 전체 실습 순서

1. 샘플 파일을 `samples` 폴더에 넣는다.
2. Codex에서 실습 폴더를 연다.
3. Codex를 플랜 모드로 전환한다.
4. 첫 프롬프트로 자동화 구조를 설계하게 한다.
5. 설계가 정리되면 구현을 요청한다.
6. 통합 결과 파일 생성 여부를 확인한다.
7. `.bat` 파일 실행까지 확인한다.

## 1단계. 샘플 파일을 `samples` 폴더에 넣기

### 학습자 할 일

- 받은 파일 3개를 `samples` 폴더에 넣는다.
- 불필요한 이전 테스트 파일은 지운다.

### 학습자 체크포인트

- 파일이 3개 모두 들어갔는가
- 파일명에 날짜와 장비명이 들어가 있는가
- 오늘 실습에서 사용할 파일만 들어 있는가

## 2단계. Codex에서 실습 폴더 열기

### 학습자 할 일

- Codex를 실행한다.
- 실습 폴더를 연다.

### 학습자 체크포인트

- 현재 폴더가 실습 폴더인지 확인한다.
- `samples`, `outputs` 폴더가 보이는지 확인한다.

## 3단계. 플랜 모드로 전환하기

### 왜 필요한가

처음부터 바로 코드를 만들게 하면 학습자가 구조를 놓치기 쉽다.  
그래서 먼저 `무엇을 자동화할지, 입력과 출력은 어디인지, 사용자는 어떻게 실행할지`를 정리하는 단계가 필요하다.

### 학습자 할 일

- Codex에서 플랜 모드로 전환한다.
- 아래 첫 프롬프트를 입력한다.

### 첫 프롬프트 예시

```text
이 폴더의 실험데이터 통합 작업을 자동화하고 싶다.
samples 폴더의 최신 실험데이터 파일 3개를 읽어서 공통 컬럼 기준으로 하나의 xlsx 파일로 통합하고,
결과는 outputs 폴더에 저장되게 해줘.
파일명 규칙도 함께 정리하고, 사용자가 나중에 쉽게 다시 실행할 수 있도록 실행 방식까지 포함해서 먼저 계획을 세워줘.
```

### 이 프롬프트의 의도

이 프롬프트는 단순히 `합쳐줘`가 아니라 아래를 함께 요구한다.

- 입력 폴더
- 파일 선택 기준
- 통합 기준
- 출력 폴더
- 재실행 방식
- 먼저 계획부터 세우기

## 4단계. 플랜 모드에서 확인할 것

Codex가 계획을 제시하면 아래 항목이 들어 있는지 확인한다.

- 어떤 파일을 읽을지
- 최신 파일 3개를 어떤 기준으로 고를지
- 공통 컬럼을 어떻게 매핑할지
- 결과 파일을 어디에 저장할지
- 사용자가 어떻게 다시 실행할지
- 설치가 필요한 것이 있는지

### 학습자 질문 예시

계획이 모호하면 아래처럼 다시 물어본다.

```text
좋다. 그런데 사용자가 PowerShell을 몰라도 실행할 수 있게 하려면 무엇이 필요해?
그리고 이 자동화에서 사용자가 미리 지켜야 할 파일명 규칙도 함께 정리해줘.
```

## 5단계. 구현 요청하기

계획이 정리되면 구현을 요청한다.

### 구현 프롬프트 예시

```text
좋다. 이제 이 계획대로 실제로 구현해줘.
samples 폴더에서 최신 csv/xlsx 파일 3개를 골라 통합하도록 만들고,
결과는 outputs 폴더에 저장되게 해줘.
사용자가 쉽게 다시 실행할 수 있도록 .bat 파일까지 만들어줘.
그리고 필요한 안내 문서도 함께 정리해줘.
```

### 구현 후 학습자가 확인할 것

- 통합 스크립트가 만들어졌는가
- 실행용 `.bat` 파일이 만들어졌는가
- 가이드 문서가 생성되거나 수정되었는가

## 6단계. 실행 전 점검 질문

구현이 끝났다고 바로 넘어가지 말고 아래를 체크한다.

- Node.js가 설치되어 있는가
- `samples` 폴더에 파일 3개가 있는가
- `outputs` 폴더가 있는가
- 파일명 규칙이 너무 제각각이지 않은가

이 단계에서 Codex에게 아래처럼 물어볼 수 있다.

```text
실행 전에 사용자가 꼭 확인해야 하는 항목을 체크리스트로 정리해줘.
```

## 7단계. 실제 실행하기

### 가장 쉬운 방법

아래 BAT 파일을 실행한다.

- `실험데이터_통합_실행.bat`

### 학습자 할 일

- BAT 파일을 더블클릭한다.
- 실행이 끝날 때까지 기다린다.

### 실행 결과로 기대하는 것

- `outputs` 폴더에 날짜가 붙은 xlsx 파일이 생성된다.

예시:

- `실험데이터_통합결과_20260421.xlsx`

## 8단계. 결과 파일 검증하기

### 학습자 할 일

- `outputs` 폴더를 연다.
- 최신 결과 파일을 연다.
- 아래 시트를 확인한다.

### 확인할 시트

1. `통합본`
2. `장비별추가항목`
3. `매핑로그`
4. `검토필요`

### 학습자 체크포인트

- 3개 파일 내용이 모두 들어갔는가
- 공통 컬럼이 정상적으로 통합되었는가
- 추가 컬럼이 보존되었는가
- `검토필요` 시트에 이상 메시지가 있는가

## 9단계. Codex에게 결과 검토 요청하기

학습자는 결과를 열어본 뒤 Codex에게 한 번 더 검토를 요청할 수 있다.

### 검토 프롬프트 예시

```text
통합 결과를 기준으로 이 자동화가 실무에서 바로 써도 되는 수준인지 점검해줘.
검토필요 시트에서 어떤 항목을 우선 확인해야 하는지도 알려줘.
```

또는 아래처럼 개선 방향을 물을 수도 있다.

```text
좋다. 이제 이 구조를 매일 자동 실행하려면 무엇을 추가하면 되는지 알려줘.
```

## 10단계. 자동화 관점에서 정리하기

실습이 끝나면 학습자가 아래 질문에 답할 수 있어야 한다.

1. 왜 프롬프트만으로는 자동화가 완성되지 않는가
2. 왜 파일명 규칙이 중요한가
3. 왜 결과 저장 위치를 미리 정해야 하는가
4. 왜 BAT 파일 같은 실행 버튼이 필요한가
5. 왜 Node.js 같은 실행 환경을 확인해야 하는가

## 강사가 학습자에게 강조할 핵심

- 자동화는 `한 번 잘 처리하는 것`이 아니라 `반복해서 같은 방식으로 돌아가는 구조`다.
- 좋은 첫 프롬프트는 작업 요청이면서 동시에 설계 요청이어야 한다.
- 처음에는 자유롭게 생각하게 하되, 실행은 단계적으로 따라가게 하는 것이 좋다.
- 최종 결과물은 코드가 아니라 `사용자가 다시 실행할 수 있는 구조`다.

## 최종 요약

이 표준 실습의 핵심은 아래 흐름을 한 번에 경험하는 것이다.

`플랜 모드로 구조를 먼저 잡는다 → 구현을 요청한다 → 파일을 실제로 통합한다 → BAT 파일로 다시 실행한다 → 결과를 검토한다`

이 흐름까지 가야 학습자는 `AI에게 요청하는 것`과 `실제로 자동화 구조를 만드는 것`의 차이를 체감할 수 있다.
