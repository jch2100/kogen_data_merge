# 06. 강사용 진행가이드

## 문서 목적

이 문서는 `실험데이터 통합 자동화` 실습을 강사가 실제 수업에서 진행할 때, 어떤 순서로 설명하고 어디에서 개입하면 좋은지 바로 참고할 수 있도록 만든 진행용 가이드다.

## 수업 목표

이 실습의 목표는 단순히 파일 3개를 합치는 것이 아니다.

- 프롬프트만으로는 자동화가 완성되지 않는다는 점을 이해한다.
- 반복 실행 가능한 자동화는 입력 규칙, 실행 방식, 출력 위치까지 필요하다는 점을 이해한다.
- ChatGPT 방식과 Codex 방식의 차이를 비교해 본다.
- Codex로 실제 자동화 흐름을 만들고, 최종적으로 `.bat` 실행 파일까지 확인한다.

## 준비물

- 구글드라이브에 올려둔 샘플 파일 3개
- 학습자 PC
- Codex 실행 환경
- Node.js 설치 환경
- 실습 폴더
  - `samples`
  - `outputs`
  - `실험데이터_통합_실행.bat`

## 전체 진행 순서

1. 샘플 파일 배포와 문제 인식
2. 파일 3개를 합치는 방법 비교
3. Codex로 자동화 실습 시작
4. 강사의 개입 포인트 정리
5. 결과 확인과 마무리

## 1. 샘플 파일 배포와 문제 인식

### 진행 방법

- 학습자들에게 구글드라이브 폴더 링크를 준다.
- 샘플 파일 3개를 다운로드하게 한다.
- 파일명은 일부러 제각각이거나 규칙이 없는 상태로 제시한다.

### 강사 질문

- 이 파일들의 문제는 무엇인가?
- 오늘 한 번 합치는 것은 가능하겠지만, 매일 반복하려면 무엇이 불편한가?
- 어떤 파일이 오늘 파일인지 어떻게 구분할 수 있을까?

### 강사 포인트

- 학습자들이 보통 `파일 형식이 다르다`, `컬럼명이 다르다`를 먼저 말한다.
- 여기서 한 걸음 더 나아가 `파일명 규칙도 중요하다`는 점을 짚어준다.
- 자동화는 내용만 맞는다고 되는 것이 아니라, 입력 파일을 기계가 안정적으로 찾을 수 있어야 한다.
- 그래서 날짜와 장비명이 들어간 파일명 규칙이 필요하다.

### 정리 멘트 예시

`한 번 수동으로 합칠 때는 파일명이 조금 엉성해도 버틸 수 있습니다. 하지만 매일 자동으로 합치려면 어떤 파일이 오늘 데이터인지 바로 구분할 수 있어야 합니다. 그래서 파일명 규칙이 자동화의 첫 단계입니다.`

## 2. 파일 3개를 합치는 방법 비교

### 진행 방법

학습자들에게 질문한다.

`이 3개 파일을 합치는 방법은 무엇일까?`

학습자 답변을 받은 뒤 아래 두 방식으로 정리한다.

### 방법 A. ChatGPT 방식

- 파일 3개를 ChatGPT에 업로드한다.
- 합쳐달라고 요청한다.
- 결과 파일을 다운로드한다.
- 오류를 점검한다.
- 원하는 폴더로 옮긴다.

### 방법 B. Codex 방식

- 파일 3개를 한 폴더에 둔다.
- Codex가 그 폴더를 기준으로 읽게 한다.
- 자동으로 병합하게 한다.
- 결과를 지정한 폴더에 저장하게 한다.
- 나중에는 `.bat` 파일로 실행을 단순화할 수 있다.

### 강사 질문

- 한 번만 할 건가, 반복해서 할 건가?
- 사람이 매번 다운로드하고 옮기는 방식이 맞는가?
- 내가 원하는 것은 “대답”인가, “실행 구조”인가?

### 강사 포인트

- ChatGPT는 일회성 처리, 탐색, 초안 만들기에 좋다.
- Codex는 파일 시스템을 기준으로 반복 실행 구조를 만들기 좋다.
- 여기서 학습자에게 `둘 중 어느 것이 더 자동화에 가까운가`를 스스로 말하게 하면 좋다.

### 정리 멘트 예시

`ChatGPT는 잘 답해주는 도구이고, Codex는 내 폴더를 기준으로 실제 실행 구조를 만드는 데 더 유리합니다. 오늘 수업은 그래서 Codex 쪽으로 가보겠습니다.`

## 3. Codex로 자동화 실습 시작

### 진행 방법

- 학습자들에게 파일 3개를 한 폴더에 저장하게 한다.
- Codex에서 해당 폴더를 열게 한다.
- 바로 정답 프롬프트를 주기 전에 먼저 묻는다.

### 강사 질문

- Codex에게 어떤 첫 문장을 주면 좋을까?
- 무엇을 입력으로 보고, 무엇을 출력으로 저장하게 해야 할까?
- 사용자는 어떻게 실행하면 가장 편할까?

### 추천 첫 프롬프트

```text
이 폴더의 최신 실험데이터 파일 3개를 읽어서 공통 컬럼 기준으로 하나의 xlsx 파일로 통합해줘. 파일명 규칙도 함께 정리하고, 결과는 outputs 폴더에 저장되게 해줘.
```

### 강사 포인트

- 학습자들이 처음에는 “합쳐줘”까지만 말하는 경우가 많다.
- 이때 강사는 자동화 프롬프트에는 아래 요소가 같이 들어가야 한다고 짚어준다.
  - 입력 기준
  - 파일 선택 기준
  - 출력 위치
  - 실행 방식
  - 검토가 필요한 경우의 처리

### 진행 팁

- 중간에 너무 빨리 손으로 고치기보다, Codex가 끝까지 어떻게 구조를 제안하는지 보게 한다.
- 처음 한 번은 완벽함보다 흐름 이해가 더 중요하다.
- 학습자에게는 “끝까지 따라가보는 것”을 추천한다.

### 정리 멘트 예시

`자동화는 한 문장으로 끝나는 것이 아니라, 그 한 문장 뒤에서 폴더 구조, 실행 방식, 결과 저장까지 이어져야 합니다. 그래서 처음 요청도 조금 더 구조적으로 해야 합니다.`

## 4. 강사의 개입 포인트

이 단계는 학습자가 중간에 막히거나, 방향이 흐려질 때 강사가 개입하는 포인트다.

### 4-1. 수정보다 새로 만드는 게 빠를 수 있다

- 기존 결과물을 조금씩 고치게 하다 보면 오히려 시간이 더 걸릴 수 있다.
- 자동화에서는 입력, 출력, 실행 구조를 새로 잡는 편이 더 빠를 때가 많다.

정리 멘트 예시:

`자동화는 수동 작업을 조금씩 고치는 것보다, 처음부터 반복 가능한 구조로 다시 짜는 편이 더 빠를 때가 많습니다.`

### 4-2. 첫 프롬프트는 플랜 모드 성격이 좋다

- 처음부터 바로 코드를 짜게 하기보다 먼저 설계를 정리하게 한다.
- 아래를 먼저 말하게 하면 좋다.
  - 무엇을 자동화할 것인가
  - 입력은 어디인가
  - 출력은 어디인가
  - 사용자는 어떻게 실행하는가

정리 멘트 예시:

`좋은 자동화는 좋은 코드보다 먼저 좋은 구조가 나와야 합니다. 그래서 첫 프롬프트는 구현 지시이면서 동시에 설계 요청이어야 합니다.`

### 4-3. 설치해야 하는 것 확인

- 이 예제는 Python이 아니라 Node.js가 필요하다.
- 먼저 `node --version`으로 설치 여부를 확인하게 한다.
- 설치가 안 되어 있으면 Node.js 공식 사이트에서 LTS 버전을 설치하게 한다.

정리 멘트 예시:

`지금 이 예제는 Node.js로 돌아갑니다. BAT 파일은 버튼이고, 실제 코드는 Node.js가 실행합니다.`

### 4-4. 구현 결과는 사용자가 쉽게 실행할 수 있어야 한다

- 최종 결과물은 코드만이 아니다.
- 사용자가 PowerShell을 몰라도 실행할 수 있는 진입점이 있어야 한다.
- 그래서 `.bat` 파일까지 만들어야 교육 효과가 높다.

정리 멘트 예시:

`자동화의 완성은 코드가 아니라 실행 경험입니다. 사용자가 더블클릭만 해도 돌아가야 진짜 실무 자동화에 가까워집니다.`

## 5. 결과 확인과 마무리

### 확인할 것

- `outputs` 폴더에 결과 파일이 생성됐는가
- 파일명이 날짜 규칙으로 저장됐는가
- `검토필요` 시트가 생성됐는가
- 사용자가 `.bat` 파일로 다시 실행할 수 있는가

### 마무리 질문

- 오늘 만든 것은 단순 병합인가, 자동화 구조인가?
- 이걸 실제 내 업무로 가져가려면 어떤 규칙을 더 정해야 할까?
- 내 업무에서는 파일명 규칙, 폴더 구조, 실행 시점 중 무엇이 가장 중요할까?

### 마무리 멘트 예시

`오늘 핵심은 파일 3개를 합친 것이 아닙니다. 내일도 같은 방식으로, 같은 규칙으로, 같은 위치에 결과가 나오게 만드는 구조를 만든 것입니다.`

## 추천 시간 배분

- 10분: 샘플 파일 배포와 문제 인식
- 10분: ChatGPT 방식과 Codex 방식 비교
- 15분: Codex 실습
- 10분: 강사 개입과 구조 정리
- 5분: 결과 확인과 마무리

총 50분 정도로 운영하기 좋다.
